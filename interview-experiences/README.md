### 1. 京东面试（一面，确认凉了）- 京东科技部门 - 云事业部门 - 技术运维（实则 AI Agent 开发工程师）

- 来源：LINUX DO
- 日期：2026-02-10
- 链接：https://linux.do/t/topic/1591661

#### 高频问题

- AI 报警服务怎么实现，AI 输入了什么数据、输出了什么结果。
- 加入 AI 之后业务指标有什么变化，准确性如何保障。
- 遇到发版、节假日等突发情况时，AI 预测失效怎么补救。
- RAG 处理过什么文件，不同类型文件如何解析。
- 遇到几个 G、几千页、图文混杂的大文件时如何实现。
- RAG 的更新频率、更新方式是手动还是自动。
- RAG 在实际业务中解决了什么问题，优化效果如何衡量。
- LangChain 在 AI 助手里的具体用法。
- AI 助手如何接入办公 OA，如何区分多条调用链路和写链路。
- MCP Server 提供了哪些工具，如何保障分析正确性。
- 对当前 AI Agent 技术发展、RAG、Skills 的看法。

#### 面试启发

- 面试官非常重视“项目闭环”，不是只问做了什么，而是追问输入、输出、指标、异常和补救机制。
- RAG 相关问题会直接落到工程细节，例如文件解析、更新策略、海量文档处理和业务效果评估。
- Agent / MCP 问题通常会继续追问到工具边界、调用链路和结果可信度。

### 2. 昨天的面试情况

- 来源：LINUX DO
- 日期：2025-05-14
- 链接：https://linux.do/t/topic/648771

#### 高频问题

- 给博物馆做 RAG 问答，如果用户问“有哪些唐代文物”，检索效果不好时怎么处理。
- 多轮对话里，前两问分别查询不同专业课程，第三问要求比较两个专业就业前景，此时如何做上下文理解和检索。
- 如果不能调用大模型做 Query Rewrite，如何在保证时延的前提下处理指代、比较、改写和检索。

#### 面试启发

- 工业级 RAG 往往不是“向量检索 + 大模型”这么简单，而是包含 FAQ 拦截、意图识别、实体提取、对话状态管理、业务规则和多路召回。
- 面试官会关注 latency 和 precision 的权衡，所以不能只说“再调一次大模型重写 Query”。
- 对于“唐代文物”这类问题，元数据过滤、标签体系、结构化字段、FAQ/Base Case 都是常见思路。
- 对于“比较前文两个实体”这类问题，需要上下文状态维护、指代消解、模板化改写或轻量模型，而不是只会把历史对话整段丢给 LLM。

#### 推荐回看章节

- `07-memory-and-rag`
- `08-tool-use`
- `10-real-world-cases`

### 3. 大模型应用面经合集（已拿 offer）

- 来源：牛客网
- 日期：2025-08-01
- 链接：https://www.nowcoder.com/feed/main/detail/0503d45073494879bbe91f5fccd6efbe

#### 高频问题

- 大模型是怎么训练出来的，Transformer 的 Encoder / Decoder 分别做什么。
- Function Calling 是怎么训练出来的，和普通生成式回答有什么差别。
- 微调方案有哪些，自己做过哪些 SFT / DPO / PEFT 相关实践。
- LangChain、AutoGen、LiteLLM 这类框架或网关为什么要用，什么时候手搓比上框架更合适。
- MCP 和 Function Calling 有什么区别，A2A 是否了解。
- ReAct、CoT、Prompt Caching、temperature / top-p / top-k 分别适合什么场景。
- RAG 的切分、多路召回、向量库选型、图数据库、幻觉治理、检索效果与回答效果如何量化。
- Workflow 的任务拆分为什么要做，拆分后如何验证效果提升。

#### 面试启发

- 牛客上的“大模型应用开发”面经已经明显从“会不会调 API”转向“是否理解整条工程链路”。
- 面试官特别喜欢把问题拆成模型层、Prompt 层、RAG 层、Workflow 层连续追问，考察知识体系是否完整。
- RAG 和 Tool Use 不再是加分项，而是很多岗位默认的必问题。

#### 推荐回看章节

- `01-fundamentals`
- `02-frameworks`
- `03-design-patterns`
- `07-memory-and-rag`
- `08-tool-use`

### 4. 29届第四范式 agent 开发实习面经（已过）

- 来源：牛客网
- 日期：2026-03-25
- 链接：https://www.nowcoder.com/feed/main/detail/77a81a03b55143c89d1caf76833676d9

#### 高频问题

- 你的 AI Agent 项目是个人开发还是团队开发，自己真正负责了哪些模块。
- 多端 Agent 应用的整体架构是什么，为什么前后端、AI 层要这样拆。
- 为什么使用 LangChain / LangGraph，以及 MCP adapter 在项目中承担什么角色。
- 常见 Agent 工作模式有哪些，例如 Tool Calling、ReAct、Reflection、Plan-and-Solve。
- 你如何理解 RAG，什么时候值得上 RAG，什么时候不值得。
- 长期记忆和短期记忆分别怎么做，上下文压缩如何设计。
- 多智能体里为什么给不同子任务分配不同模型，如何权衡 token 成本与质量。
- 工具调用失败或基础模型卡死时，有没有熔断、重试、集群切换等保底机制。

#### 面试启发

- 面试官会沿着项目真实实现一路追到“架构拆分、节点设计、异常兜底”，非常容易识别出项目是否只是包装。
- Memory、RAG、模型路由、熔断机制这几个点，都是现在 Agent 岗非常爱深挖的工程话题。
- “做过多端 Agent”并不够，必须能说清工具层、确认节点、数据库交互和写操作保护策略。

#### 推荐回看章节

- `02-frameworks`
- `03-design-patterns`
- `04-system-design`
- `07-memory-and-rag`
- `08-tool-use`

### 5. 百度秋招智能体算法二面

- 来源：牛客网
- 日期：2025-11-11
- 链接：https://www.nowcoder.com/feed/main/detail/6fd55b055eba4901a7e5b8d37b9a6ba1

#### 高频问题

- Agent 系统的 Prompt 是怎么设计和迭代的，有没有做过自动 Prompt 优化。
- 用户请求不完整时，如何补全意图并保持系统稳健。
- LangChain 的 memory 在多用户并发场景下如何隔离，如何保证线程安全和状态一致性。
- SFT、DPO、偏好对构造、训练样本清洗这些微调问题是否做过。
- 为什么使用 vLLM，KV cache 如何帮助推理加速，自己做过哪些服务优化。
- 如果既要支持 Streaming 输出，又面临高延迟，系统如何折中设计。
- 多轮对话上下文状态如何管理，高并发下一致性如何保证。
- 工具调用链上的超时、故障、反馈策略以及 Agent 模块监控与定位怎么做。

#### 面试启发

- 智能体算法岗并不会只问“算法”，会把你拉回到 Prompt、服务部署、并发状态管理和容错设计。
- “线程安全”“高并发一致性”“Streaming 延迟折中”这类问题说明 Agent 岗越来越偏工程化。
- 面试官默认你不仅会做 demo，还要会解释线上服务怎么跑、出问题怎么查。

#### 推荐回看章节

- `04-system-design`
- `05-coding`
- `07-memory-and-rag`
- `08-tool-use`
- `09-evaluation-and-safety`

### 7. 贝壳找房 RAG 面经

- 来源：牛客网
- 日期：2025-11-20
- 链接：https://www.nowcoder.com/feed/main/detail/b8c08e8aa1974639965a795a2d7f82d8

#### 高频问题

- RAG 和 Agent 项目是否真实做过，为什么选当前方案而不是其他方案。
- Transformer 架构、多头注意力复杂度、DPO 全流程是否扎实。
- RAG 的整体流程、检索链路和效果优化有没有做过量化。
- 项目里的指标、数据规模、瓶颈和真实 trade-off 能不能讲清楚。
- 手撕代码和基础算法依然会出现，不能把岗位理解成纯“配流程”工作。

#### 面试启发

- RAG 岗经常是“项目深挖 + 基础模型原理 + 手撕代码”三合一，不是只问向量库。
- 面试官很看重“做过”和“理解过”的差别，特别会追问为什么做 A 不做 B。
- 如果简历上挂了 RAG / Agent 项目，最好提前准备量化指标、数据规模和失败 case。

#### 推荐回看章节

- `01-fundamentals`
- `05-coding`
- `07-memory-and-rag`
- `09-evaluation-and-safety`

### 10. 面试官：关于 RAG，你了解多少？（高频追问整理）

- 来源：牛客网
- 日期：页面未展示完整日期
- 链接：https://www.nowcoder.com/feed/main/detail/4878c1bc11664645817be072e5a0b0b7

#### 高频问题

- 实际项目里除了标准 RAG 流程，还有哪些优化技巧，例如多路召回、重排序、模型微调。
- 生成环节如何评估效果，除了 Rouge 还会不会看人工评估、资源效率和线上指标。
- RAG 为什么会出现幻觉，幻觉来自检索链路还是生成链路。
- 如何减少幻觉，知识图谱、ReAct、自我纠偏、兜底拒答各适合什么场景。
- 边界 case 怎么处理，比如知识库没有答案、时间敏感问题、格式错误、输出不可解析。

#### 面试启发

- 这类问答帖反映出 RAG 高频追问已经从“是什么”升级到“怎么评估、怎么优化、怎么兜底”。
- 面试官很爱围绕幻觉、边界 case、拒答策略继续追问，说明线上稳健性是核心考点。
- 如果只会背标准流程，很容易在优化和评测问题上被继续追问。

#### 推荐回看章节

- `07-memory-and-rag`
- `09-evaluation-and-safety`

## 使用建议

1. 先通读面经里的提问方式，观察面试官是如何从“做过项目”追问到“是否真正上线过”的。
2. 再回到题库中按章节补知识点，不要只记结论，要准备输入、输出、指标、异常、降级、成本和时延这些工程化回答。
3. 如果后续继续收录面经，优先补充来源、日期、原始链接、核心问题和对应复习章节。
